## 2. 设备绑定与鉴权（AUTH）

### FR-AUTH-001 设备首次开机激活

- **优先级**：P0 / **状态**：📝
- **角色**：父母 / 手表
- **用户故事**：作为父母，当我第一次给手表上电时，手表显示二维码（含设备唯一 ID），我用 APP 扫描后输入手机验证码即可完成绑定
- **触发**：首次开机或恢复出厂后
- **验收标准**：
  - [ ] 手表屏幕显示二维码 ≤ 5 秒内
  - [ ] APP 扫码后向后端发起 `POST /api/v1/devices/bind`，返回 `device_token` 写入手表
  - [ ] 绑定完成后手表自动跳转表盘
  - [ ] 同一台手表只能绑定一个"主家庭"（解绑需主家长在 APP 端确认）

### FR-AUTH-002 多家长协同管理

- **优先级**：P0 / **状态**：📝
- **角色**：主家长 / 副家长
- **用户故事**：作为主家长（爸爸），我可以邀请妈妈、奶奶加入"家庭组"，副家长能看位置、收消息、拨打手表，但不能解绑设备或修改 SOS 号码
- **验收标准**：
  - [ ] 主家长可在 APP 中生成 24h 有效的邀请码
  - [ ] 副家长输入邀请码后加入家庭组
  - [ ] 副家长权限：读位置、收发微聊、拨打 ✅；解绑、改 SOS、改围栏 ❌
  - [ ] 一个家庭组上限 5 个家长

### FR-AUTH-003 手表端密码保护（防误操作）

- **优先级**：P1 / **状态**：📝
- **用户故事**：作为父母，我希望孩子在手表上点击"系统设置 > 恢复出厂"时需要 4 位 PIN 才能继续
- **验收标准**：
  - [ ] 进入"系统设置"页面需输入 PIN
  - [ ] PIN 在 APP 端设置，默认 `2024`
  - [ ] 输错 5 次锁定 5 分钟

### FR-AUTH-004 设备离线场景的会话延续

- **优先级**：P0 / **状态**：📝
- **用户故事**：作为父母，即使手表临时断网，重新联网后 APP 中的历史消息和位置都能补齐
- **验收标准**：
  - [ ] `device_token` 使用 JWT，有效期 30 天，到期前自动刷新
  - [ ] 手表离线时本地缓存事件（位置/录音），上线后批量上报
  - [ ] 后端按事件 `timestamp` 排序入库，APP 时间线正确

## 3. 通话（CALL）

### FR-CALL-001 拨打白名单号码

- **优先级**：P0 / **状态**：📝
- **角色**：孩子
- **用户故事**：作为孩子，我在通讯录中点击"妈妈"头像，手表立即拨打妈妈的手机
- **触发**：在通讯录页面点击联系人
- **验收标准**：
  - [ ] 通讯录最多 10 个联系人（足够覆盖直系亲属与老师）
  - [ ] 点击后 2 秒内开始振铃
  - [ ] 通话界面显示对方头像 + 通话时长 + 静音/挂断按钮
  - [ ] 通话音量默认 60%，按物理音量键可调
  - [ ] 通话结束后回到表盘，记录通话历史

### FR-CALL-002 接听来电（白名单内）

- **优先级**：P0 / **状态**：📝
- **用户故事**：作为孩子，当妈妈打来电话时，手表振动 + 振铃 + 屏幕显示头像，我点绿色按钮接听
- **验收标准**：
  - [ ] 仅白名单号码的来电会显示和振铃
  - [ ] 非白名单号码：手表完全不响（无 UI 提示），但 APP 端记录"来过陌生电话"
  - [ ] 振铃音量 ≥ 60dB（在儿童手表上能听清）
  - [ ] 接通后双向音质 PESQ ≥ 3.0

### FR-CALL-003 通讯录管理（家长 APP 主导）

- **优先级**：P0 / **状态**：📝
- **用户故事**：作为父母，我在 APP 中维护手表的通讯录，添加 / 删除联系人后下次手表上线自动同步
- **验收标准**：
  - [ ] APP 中编辑联系人姓名、手机号、头像、备注（爸爸/妈妈/老师等）
  - [ ] 修改后 MQTT 下行同步到手表，30 秒内生效
  - [ ] 头像在手表端预渲染为 80×80 PNG 节省渲染时间

### FR-CALL-004 通话历史

- **优先级**：P1 / **状态**：📝
- **用户故事**：作为父母，我在 APP 中能看到孩子最近 30 天的通话记录（谁打的、几点、时长）
- **验收标准**：
  - [ ] 手表端在本地存最近 100 条记录（用 NVS）
  - [ ] 每次连接 MQTT 时同步增量记录到后端
  - [ ] APP 展示通话方向（拨入/拨出/未接）、对方头像、时长

### FR-CALL-005 通话静音与拒接

- **优先级**：P1 / **状态**：📝
- **用户故事**：作为孩子，我能在通话中按"静音"键暂时让对方听不到我的声音，或主动挂断
- **验收标准**：
  - [ ] 通话界面有"静音"、"挂断"两个大按钮（≥ 70×70 px）
  - [ ] 静音状态下手表麦克风不采集，对方听到背景静音
  - [ ] 挂断后立即停止 4G 计费时段

### FR-CALL-006 通话期间防误触

- **优先级**：P1 / **状态**：📝
- **用户故事**：作为孩子，通话中我把手腕放下、衣服蹭到屏幕时不应误挂断
- **验收标准**：
  - [ ] 通话期间触摸屏的"挂断"按钮需长按 0.5 秒才触发
  - [ ] 抬腕/低头检测（IMU）期间屏幕自动锁触
  - [ ] 物理按键挂断（如果有）需双击触发

## 4. 定位与轨迹（LOC）

### FR-LOC-001 实时位置上报

- **优先级**：P0 / **状态**：📝
- **用户故事**：作为父母，我在 APP 地图上看到孩子当前位置，最新位置不超过 1 分钟前
- **触发**：
  - 手表移动 > 50 米
  - 距上次上报 ≥ 60 秒（移动中）/ ≥ 5 分钟（静止）
  - APP 主动请求"刷新位置"
- **验收标准**：
  - [ ] 定位精度 ≤ 10m（室外开阔）/ ≤ 50m（城市峡谷）
  - [ ] MQTT topic：`watch/{device_id}/location/up`
  - [ ] 数据格式：`{lat, lng, accuracy, speed, heading, timestamp, source: gnss/lbs/wifi}`

### FR-LOC-002 GNSS + 基站 + WiFi 混合定位

- **优先级**：P0 / **状态**：📝
- **用户故事**：作为父母，即使孩子在室内（GPS 失效），我也能看到大致位置（小区级别）
- **验收标准**：
  - [ ] 户外优先用 EC800N 的 GNSS（GPS + 北斗 + GLONASS + Galileo）
  - [ ] 室内回退到 LBS 基站定位（精度 50-500m）
  - [ ] WiFi 定位（可选，V1.1）：扫描周围 BSSID 上报，后端调高德 WiFi 定位 API
  - [ ] APP 中标识定位来源（GPS/LBS/WiFi）以告知精度

### FR-LOC-003 历史轨迹

- **优先级**：P1 / **状态**：📝
- **用户故事**：作为父母，我能在 APP 中查看任意日期的孩子轨迹（折线 + 时间点）
- **验收标准**：
  - [ ] 后端保留 90 天轨迹（约 90 天 × 200 点/天 × 100B = 1.8MB/孩子）
  - [ ] APP 在地图上以折线展示，关键点（>10 分钟停留）打标
  - [ ] 支持选择日期范围（最长 30 天）

### FR-LOC-004 位置上报频率自适应

- **优先级**：P1 / **状态**：📝
- **用户故事**：作为父母，我希望孩子在学校时位置上报频率降低（省电），放学路上自动加密
- **验收标准**：
  - [ ] 静止 ≥ 5 分钟 → 切换"驻留模式"，每 10 分钟上报一次
  - [ ] 检测到位移 > 20m → 切换"移动模式"，每 60 秒上报
  - [ ] 在电子围栏内（家/学校）→ 每 5 分钟上报
  - [ ] APP 端可手动覆盖：高频（30 秒）/ 标准 / 省电

### FR-LOC-005 APP 主动请求即时定位

- **优先级**：P0 / **状态**：📝
- **用户故事**：作为父母，我点 APP 中"立即定位"，手表 30 秒内返回最新位置
- **验收标准**：
  - [ ] APP 通过 MQTT 下发 `cmd/locate` 命令
  - [ ] 手表收到后立即唤醒 GNSS 模块，30 秒内返回
  - [ ] 失败时返回最近一次的缓存位置 + 失败原因（信号差/GPS 关闭/无电）

### FR-LOC-006 离线位置缓存

- **优先级**：P1 / **状态**：📝
- **用户故事**：作为父母，孩子在地下停车场断网 1 小时后回到地面，APP 应能拿到地下时的最后位置
- **验收标准**：
  - [ ] 手表本地存最近 24 小时位置（用 LittleFS，约 1440 条 × 100B ≈ 150KB）
  - [ ] 重新连上 MQTT 时批量上报增量
  - [ ] 后端按 timestamp 去重入库

## 5. SOS 紧急呼叫（SOS）

### FR-SOS-001 长按按键触发

- **优先级**：P0 / **状态**：📝
- **用户故事**：作为孩子，遇到紧急情况长按右侧"SOS"按钮 3 秒，手表自动呼叫家长，并将我的位置发给所有家长
- **验收标准**：
  - [ ] 按键需独立物理键，不复用电源/Home（位置在表壳侧面，有凸起触感）
  - [ ] 按下 3 秒进入"SOS 倒计时 5 秒"界面，振动强提示
  - [ ] 倒计时期间松手取消（防误触）；不松手则触发
  - [ ] 触发后：① 立即上报位置到 APP（推送 + 短信）② 依次拨号 SOS 列表 ③ 录音 30 秒上传

### FR-SOS-002 SOS 号码列表配置

- **优先级**：P0 / **状态**：📝
- **用户故事**：作为父母，我在 APP 中配置 3 个 SOS 号码（爸爸 → 妈妈 → 奶奶），手表按顺序呼叫
- **验收标准**：
  - [ ] APP 设置 SOS 列表（最多 3 个，至少 1 个）
  - [ ] 手表拨号失败（无人接 / 关机 / 占线 60 秒）后自动呼叫下一个
  - [ ] 所有号码都失败时进入"等待 60 秒后从头重试"循环（最多 3 轮）

### FR-SOS-003 上课禁用期间 SOS 仍可用

- **优先级**：P0 / **状态**：📝
- **用户故事**：即使在"上课禁用"时段，SOS 也必须能用
- **验收标准**：
  - [ ] 上课禁用屏蔽通讯录通话与微聊，但**不屏蔽** SOS 按键
  - [ ] SOS 触发后强制覆盖任何其他状态（包括充电中、关机倒计时中）

## 6. 微聊语音消息（IM）

### FR-IM-001 手表按住录音发送给家长

- **优先级**：P0 / **状态**：📝
- **用户故事**：作为孩子，我在"微聊"页面按住录音键说话，松手后消息发给爸爸妈妈
- **验收标准**：
  - [ ] 录音页面有大头像（按住）+ 实时音量波形
  - [ ] 录音最长 30 秒（防误按一直录）
  - [ ] 用 Opus 编码 16kbps，30 秒约 60KB
  - [ ] 上传走 HTTPS `POST /api/v1/im/voice`，返回 `msg_id`
  - [ ] 后端通过 MQTT `app/{family_id}/im/down` 推送给所有家长 APP

### FR-IM-002 家长 APP 给手表发语音

- **优先级**：P0 / **状态**：📝
- **用户故事**：作为父母，我在 APP 按住录音键说话，手表 5 秒内收到消息，振动提示
- **验收标准**：
  - [ ] APP 录音 + Opus 编码 + 上传后端
  - [ ] 后端 MQTT 下行到手表，手表本地存到 LittleFS
  - [ ] 手表点击播放：QMI8658 检测抬腕亮屏，自动播放最新一条
  - [ ] 已读状态回传 APP（已读/未读标识）

### FR-IM-003 微聊列表与历史

- **优先级**：P1 / **状态**：📝
- **用户故事**：作为父母，我在 APP 中能看到与孩子的完整聊天历史
- **验收标准**：
  - [ ] APP 中按时间倒序展示，每条显示发送者头像 + 时长 + 时间
  - [ ] 后端保留 180 天，老消息归档到冷存储
  - [ ] 手表端只保留最近 50 条（NVS 容量限制）

### FR-IM-004 群聊（家庭组）

- **优先级**：P1 / **状态**：📝
- **用户故事**：作为家长，我和配偶在一个家庭组里，手表发的消息我们俩都能收到
- **验收标准**：
  - [ ] 微聊是"家庭组级别"的 group chat，不是 1-1
  - [ ] 任一家长发消息，组内其他家长和手表都能收到
  - [ ] 谁先听完消息后，"已读"对所有人可见

### FR-IM-005 文字消息（家长发给手表）

- **优先级**：P2 / **状态**：❄️
- **用户故事**：4 岁孩子识字有限，V1 不做文字消息

## 8. 表盘与设置（UI）

### FR-UI-001 默认表盘

- **优先级**：P0 / **状态**：📝
- **用户故事**：作为孩子，默认看到一个卡通女孩主题的表盘，显示时间、电量、信号
- **验收标准**：
  - [ ] 时间字体大、易读（4 岁孩子开始学认数字）
  - [ ] 角色形象配合时段表情变化（早/中/晚不同表情）
  - [ ] 电量 < 20% 时图标变红
  - [ ] 4G 信号格 0-4 格显示

### FR-UI-002 主菜单与导航

- **优先级**：P0 / **状态**：📝
- **用户故事**：作为孩子，在表盘上下滑/上滑/左滑/右滑可以快速进入功能
- **验收标准**：
  - [ ] 下滑：通知中心（未读微聊、未接电话）
  - [ ] 上滑：快捷设置（音量、亮度、勿扰）
  - [ ] 左滑：相机
  - [ ] 右滑：通讯录
  - [ ] 点击表盘：进入主菜单九宫格（拨号 / 微聊 / 相机 / 相册 / 计步 / 闹钟 / 设置 / 我 / 应用 待扩展）

### FR-UI-003 抬腕亮屏

- **优先级**：P0 / **状态**：📝
- **用户故事**：作为孩子，手表自动亮屏（不用按按键），方便看时间
- **验收标准**：
  - [ ] QMI8658 检测抬腕动作（角度变化阈值可调）
  - [ ] 亮屏后 5 秒无操作自动熄屏
  - [ ] 误触率 < 5%（夜里转身不应频繁亮屏）
  - [ ] 上课禁用期间禁用抬腕亮屏

### FR-UI-004 设置项（手表端）

- **优先级**：P0 / **状态**：📝
- **用户故事**：作为孩子，我能调节亮度、铃声音量、振动开关
- **验收标准**：
  - [ ] 亮度：4 档（25/50/75/100%）
  - [ ] 音量：通话音量、铃声音量、媒体音量分别 5 档
  - [ ] 振动：开/关
  - [ ] 其他设置（围栏、SOS 号码等）锁定，进入需 PIN

### FR-UI-005 表盘自定义（V2 准备）

- **优先级**：P2 / **状态**：❄️

## 14. OTA 与远程管理（OTA）

### FR-OTA-001 双分区 A/B OTA

- **优先级**：P0 / **状态**：📝
- **用户故事**：作为开发者，我推送新固件后所有手表能在夜间充电时自动升级
- **验收标准**：
  - [ ] ESP32-S3 双 OTA 分区（ota_0、ota_1，各 6MB）
  - [ ] 下载新固件到非运行分区
  - [ ] 验证签名（Ed25519 公钥编入 bootloader）通过后切换分区重启
  - [ ] 启动后回滚检测：30 秒内未"提交"则回滚

- **优先级**：P1 / **状态**：📝
- **验收标准**：
  - [ ] 支持 bsdiff 算法，常规升级包 ≤ 500KB（节省 4G 流量）
  - [ ] 服务端按版本对生成差分包

### FR-OTA-003 触发策略

- **优先级**：P0 / **状态**：📝
- **验收标准**：
  - [ ] 默认：电量 ≥ 50% 且充电中且 04:00-06:00 期间自动检测升级
  - [ ] 手动：APP 中"立即升级"
  - [ ] 强制：服务端可标记 `force=true`，绕过条件
